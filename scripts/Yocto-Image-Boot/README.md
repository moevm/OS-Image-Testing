## Сборка Yocto образа через докер

Скрипт направлен на базовую разовую инициализацию тома докера и последующую многоразовую сборку образа. Самые объёмные данные сохраняются и передаются через том, остальные копируются из среды пользователя.

Самую первую сборку необходимо произвести вручную (командами терминала), чтобы по итогу появилась папка `poky/build/` с результатами сборки.

Затем необходимо запустить скрипт `init_volume.sh`, чтобы поместить самую тяжёлую папку с артефактами и кэшем сборки в том для последующего переноса в докер.

`docker compose up --build` запустит сборку через докер на основе текущего содержания папки `poky/` за исключением `build/`, которая подаётся отдельно через том.

**!** Для корректной работы путь при первой ручной сборке должен совпадать с путём рабочей директории в докере (для сохранения путей в конфигурационных файлах).

**!!** Первая сборка через докер будет дольше, определённые пакеты необходимо подтянуть повторно.

**!!!** Некоторые файлы возможно придётся подредактировать из контейнера/среды хоста, в зависимости от ошибок.
