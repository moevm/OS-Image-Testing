# Готовые инструменты и решения для тестирования образов Yocto Project, API

Для yocto project существует множество готовых решений для взаимодействия и тестирования.

Большинство из них требуют изменения конфигурационного файла перед сборкой образа `local.conf` в директории `/build`, путём добавления в него информации для сборки (иногда требуется дополнительные действия).

### Встроенные механизмы тестирования

- Обширная библиотека тестов и возможности для их кастомизации (`bitbake-selftest`, `oe-selftest`, `testimage`... [подробнее](https://docs.yoctoproject.org/test-manual/intro.html#how-tests-map-to-areas-of-code)),

- Ptest (Package test) для тестирования целостности и функциональности различных пакетов [подробнее](https://docs.yoctoproject.org/test-manual/ptest.html),

- Python API (pytests) для создания собственных тестов, на основе Ptest.

### Запуск тестов

#### Автоматический режим

Для запуска автоматического тестирования необходимо:

1. Изменить конфигурацию `local.conf`

```
TESTIMAGE_AUTO = "1"
```

2. Собрать образ

```bash
bitbake image
```

#### Ручной режим

Для ручного тестирования необходимо:

1. Изменить конфигурацию `local.conf`

```
IMAGE_CLASSES += "testimage"
```

2. Собрать образ

```bash
bitbake image
```

3. Запустить тесты

```bash
bitbake -c testimage image
```

#### С помощью Ptest

Для запуска тестирования отдельных пакетов можно воспользоваться встроенным решением через **Ptest**. Для этого необходимо:

1. Изменить конфигурацию `local.conf`

```
DISTRO_FEATURES:append = " ptest"
EXTRA_IMAGE_FEATURES += "ptest-pkgs"
```

2. Собрать образ

```bash
bitbake image
```

3. Запустить тесты

```bash
ptest-runner
```

### Стресс-тесты

Для стресс-тестирования (Performance или Endurance) можно воспользоваться утилитой **stress-ng**. 

Подключается в конфигурационном файле:

```
IMAGE_INSTALL:append = " stress-ng"
```

Данная утилита позволяет запускать настраиваемые нагрузки на различные элементы устройства, выводить метрики и взаимодействовать с ptest в скриптах.

Пример работы

```bash
stress-ng --cpu 0 --cpu-method all -t 10m
```

### Анализ производительности

Для непосредственного анализа работы устройства и его комплектующих существует утилита **perf**.

Подключение аналогичным образом:

```
IMAGE_INSTALL:append = " perf"
```

Эта утилита специализируется на анализе и обработке производительности устройства. Она способна профилировать компоненты устройства для определения уровня потребления ресурсов процессов и функций, выводить статистику и создавать визуализацию с помощью надстройки Flame Graphs.

Пример работы

```bash
perf stat ls
```

### Autobuilder

Это инструмент, используемый в Yocto Project для автоматизации процесса сборки и тестирования

Что можно делать:
* использовать различные типы тестов, такие как ptest, testimage, или даже сторонние инструменты, такие как stress-ng и perf
* можно указать, какие тесты должны быть запущены на собранном образе. Это может включать как встроенные тесты Yocto, так и ваши собственные скрипты
* результаты тестирования будут доступны через веб-интерфейс Buildbot, что позволяет отслеживать успех или неудачу тестов
