# Готовые инструменты и решения для тестирования образов Yocto Project, API

Для yocto project существует множество готовых решений для взаимодействия и тестирования.

Большинство из них требуют изменения конфигурационного файла перед сборкой образа `local.conf` в директории `/build`, путём добавления в него информации для сборки (иногда требуется дополнительные действия).

### Встроенные механизмы тестирования

- Обширная библиотека тестов и возможности для их кастомизации (`bitbake-selftest`, `oe-selftest`, `testimage`... [подробнее](https://docs.yoctoproject.org/test-manual/intro.html#how-tests-map-to-areas-of-code)),

- Ptest (Package test) для тестирования различных пакетов [подробная информация и использование](https://docs.yoctoproject.org/test-manual/ptest.html),
- Python API (pytests) для создания собственных тестов, на основе Ptest.

### Запуск тестов

#### Автоматический режим

Для запуска автоматических тестов, необходимо изменить конфигурацию local.conf и собрать образ:

```
TESTIMAGE_AUTO = "1"
```

```bash
bitbake image
```

#### Ручной режим

Для ручного тестирования инструкции аналогичные:

```
IMAGE_CLASSES += "testimage"
```

```bash
bitbake -c testimage image
```

### Стресс-тесты

Для стресс-тестирования (Performance или Endurance) можно воспользоваться утилитой stress-ng, которая также подключается в конфигурационном файле:

```
IMAGE_INSTALL:append = " stress-ng"
```

Данная настройка позволяет запускать настраиваемые нагрузки на различные элементы устройства, выводить метрики и взаимодействовать с ptest в скриптах.
