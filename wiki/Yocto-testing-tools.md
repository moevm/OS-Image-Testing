# Инструменты Yocto для тестирования



## Ptest (Package Test)
   * механизм тестирования пакетов, построенных с помощью OpenEmbedded
   * ptest позволяет запускать тесты на целевой машине для проверки функциональности пакетов
   * хотя обычно используется для функционального тестирования, может быть использован и для тестирования производительности

1. Добавьте в local.conf

```
IMAGE_FEATURES += "ptest"
```

2. Соберите образ:
```
bitbake <image>
```

3. На целевой системе выполните:

```
ptest-runner
```


## OEQA Runtime Tests (testimage)
   * фреймворк для создания и запуска тестов на целевой системе
   * позволяет писать тесты на Python, которые могут измерять производительность программ

1. Добавьте в local.conf 

```
IMAGE_CLASSES += "testimage"
TESTIMAGE_AUTO = "1"
```

2. Соберите образ:
```
bitbake <image>
```

3. Запустите тесты

```
bitbake -c testimage <image>
```

## stress-ng
* инструмент для создания нагрузки на систему
* позволяет тестировать производительность CPU, памяти, ввода/вывода, сети и других компонентов

1. Добавьте в local.conf 

```
IMAGE_INSTALL:append = " stress-ng"
```

2. Соберите образ:
```
bitbake <image>
```

3. На целевой системе выполните

```
stress-ng --cpu 4 --timeout 60s
```



## perf
* инструмент для профилирования производительности системы
* анализирует производительность CPU, памяти, ввода/вывода и других компонентов

1. Добавьте в local.conf 

```
IMAGE_INSTALL:append = " perf"
```

2. Соберите образ:
```
bitbake <image>
```

3. На целевой системе выполните

```
perf stat -e cycles,instructions ./your_binary
```

## Autobuilder
* инструмент, используемый в Yocto Project для автоматизации процесса сборки и тестирования

Что можно делать:
* использовать различные типы тестов, такие как ptest, testimage, или даже сторонние инструменты, такие как stress-ng и perf
* можно указать, какие тесты должны быть запущены на собранном образе. Это может включать как встроенные тесты Yocto, так и ваши собственные скрипты
* результаты тестирования будут доступны через веб-интерфейс Buildbot, что позволяет отслеживать успех или неудачу тестов


Для тестирования производительности образов Yocto Project доступны различные инструменты, такие как ptest, OEQA Runtime Tests, stress-ng, perf и Autobuilder. Эти инструменты предлагают широкий спектр возможностей, позволяя гибко настраивать процесс тестирования, и удовлетворяют большинству потребностей в оценке производительности.
