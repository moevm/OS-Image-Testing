# Утилиты для performance тестирования 

## Какие метрики можно анализировать


Связанные с дисковой подсистемой:
* IOPS (Input/Output Operations Per Second) - количество операций чтения/записи в секунду
* Latency - время выполнения запроса
* Queue Depth - количество операций ввода-вывода, ожидающих выполнения

Связанные с сетевой подсистемой:
* Network Throughput - пропускная способность, скорость передачи данных по сети
* Packet Latency - время передачи пакета от отправителя к получателю
* Packet Loss - процент потерянных пакетов при передаче данных
* Jitter - вариации задержки пакетов данных при их передаче по сети. Измеряется в миллисекундах (мс) и отражает изменчивость времени задержки пакетов данных при их передаче между двумя точками сети


Связанные с системными вызовами:
* Latency - сколько времени занимает выполнение одного системного вызова
* Throughput - пропускная способность
* call count - общее число вызовов конкретного системного вызова
* user time - время, проведенное в пользовательском пространстве
* system time - время проведенное в ядре
* error rate - процент неудачных вызовов


Связанные с IPC:
* Latency - время на выполнение одной операции IPC
* Throughput - пропускная способность выполнения IPC-операций
* Call count - общее число выполненных операций IPC
* Error rate - процент неудачных или ошибочных операций IPC 
* CPU utilization - загрузка процессора при работе IPC


Связанные с виртуальной памятью:
* Page Fault Rate - количество обращений к страницам, отсутствующим в оперативной памяти. Различают soft page faults, когда страница уже есть в RAM, но не отображена в адресном пространстве процесса, и hard page faults, требующие загрузки страницы с диска
* Memory Throughput - скорость операций выделения и освобождения памяти
* Latency обработки page faults - время, необходимое системе для обработки page fault
* Fragmentation - степень разрозненности свободных блоков памяти, влияющая на эффективность выделения больших непрерывных блоков
* Context Switches - количество переключений между процессами

## Утилиты 


Для метрик дисковой системы:
1. FIO (Flexible I\O Tester)
   * Метрики: IOPS, Throughput, Latency, Queue Depth
   * Создает потоки ввода/вывода данных с диска, чтобы оценить производительность файловой системы Linux
   * Утилита посчитает IOPS и пропускную способность системы, а также позволит оценить глубину очереди операций ввода/вывода. Утилита работает со специальными файлами, в которых прописываются настройки и условия теста. 
   * FIO агрегирует данные, суммируя результаты из нескольких потоков, что позволяет получить общую картину производительности системы.
   * Пример содержимого файла 
   
     ```
     [global]
          name=fio-rand-read
          filename=fio-rand-read
          rw=randread
          bs=4K
          direct=0
          numjobs=1
          time_based=1
          runtime=900
     ```
     
     Этот конфигурационный файл задаёт тест, который в течение 15 минут выполняет случайные операции чтения блоками по 4 КБ из файла fio-rand-read одним потоком, используя не прямой ввод-вывод (кэш ОС). 
   * Может быть добавлена в образ Yocto через bitbake или установлен на целевую систему после запуска
2. ioping
   * Метрики: latency
   * Простой инструмент для мониторинга задержек ввода-вывода диска в режиме реального времени
   * ioping выводит статистику по задержкам, позволяя оценить среднюю и максимальную задержку
   * Параметры:
     * `-c count` (остановить после указанного количества запросов)
     * `-w deadline` (остановить после указанного количества времени)
     * `-p period` (отображать raw статистику после каждого указанного количества запросов)
     * `-P period` (отображать raw статистику после каждого указанного количества секунд)
     * `-i interval` (интервал между запросами в секундах)
     * `-s size` (размер запроса)
     * `-S size` (размер рабочего набора)
   * Может быть добавлена в образ Yocto через bitbake


Связанные с сетью:
1. iperf
   * Метрики: Network Throughput, Packet Latency, Packet Loss (UDP), Jitter (UDP)
   * Инструмент для тестирования сети, который может создавать потоки данных TCP и UDP и измерять пропускную способность сети, по которой они передаются. Для UDP тестов можно оценить packet loss.
   * Может быть добавлена в образ Yocto через bitbake
   * iperf агрегирует данные, вычисляя среднюю пропускную способность и процент потерянных пакетов за тестовый период
   * Параметры:
     * `-s` (Запускает утилиту в режиме сервера, ожидая подключения клиента)
     * `-c IP` (Запускает утилиту в режиме клиента, подключаясь к серверу по указанному IP)
     * `-u` (Включает тестирование UDP вместо TCP)
     * `-P` (Указывает количество параллельных потоков для тестирования)
     * `-t` (Устанавливает длительность теста в секундах)
   * Вывод включает Bandwidth (Пропускная способность (в Mbps)), Packet Loss (Процент потерянных пакетов), Jitter (Вариации задержки (в мс))
     
     Пример вывода
   
     ```
      [ ID] Interval       Transfer     Bandwidth       Jitter   Lost/Total Datagrams
      [  3] 0.0-10.0 sec   1.25 MBytes   1.05 Mbits/sec   0.123 ms  5/1000 (0.5%)
     ```
2. hping
   * Метрики: Packet Latency, Packet Loss
   * Утилита для генерации и анализа сетевых пакетов на основе TCP/IP. Она позволяет отправлять кастомизированные пакеты (ICMP, TCP, UDP) и анализировать их отклик
   * Может быть добавлена в образ Yocto через bitbake
   * Параметры:
     * `--icmp` (Отправка ICMP эхо-запросов)
     * `--udp` (Отправка UDP пакетов)
     * `--interval` (Интервал между отправкой пакетов)
     * `--count число (-c)` (Количество отправляемых пакетов)
     * `--fast` (Отправка 10 пакетов в секунду)
     * `--flood` (Максимальная скорость отправки пакетов)
   * Вывод включает статистику отправленных и полученных пакетов, а также время отклика (RTT). Для оценки джиттера необходимо вручную анализировать разницу между временем отклика каждого пакета.


Связанные с системными вызовами:
1. strace
   * Метрики: Latency, call count, error rate, system time
   * Инструмент для трассировки системных вызовов процесса. Позволяет измерять время выполнения каждого вызова (время в ядре), собирать агрегированную статистику по количеству вызовов, суммарному и среднему времени, а также количеству ошибок.
   * Может быть установлен в образ Yocto через bitbake 
   * Параметры:
     * `-T` (показывает время выполнения каждого системного вызова)
     * `-c` (ведет счет ошибок, вызовов и время на выполнение всех системных вызовов)
     * `-S` (сортирует информационные блоки, которые выводятся с помощью параметра -c)
     * `-e` (позволит фильтровать определенные системные вызовы, а также события)
     * `-P` (отслеживает исключительно системные вызовы, которые относятся к указанному пути)
     * `-v` (позволит вывести дополнительные блоки информации)

2. LTP 
   * Метрики: Latency, throughput, error rate
   * Набор комплексных тестов для системных вызовов и IPC, позволяющий проводить стресс-тестирование и проверять стабильность работы syscall-интерфейса под нагрузкой
   * Выводит отчёты по каждому тесту, включая количество успешных и неудачных вызовов, время выполнения
   * Добавление в образ через bitbake
   * Пример запуска 
     
     ```bash
     runltp -f syscalls -s open,read,write
     ```
3. stress-ng
   * Метрики: throughput, error rate
   * инструмент для создания искусственной нагрузки на различные подсистемы Linux, включая системные вызовы
   * Пример запуска 
     
     ```bash
     stress-ng --fork 4 --timeout 60s
     ```
4. perf
   * perf stat - собирает общую статистику по системным вызовам и аппаратным событиям во время выполнения программы, сколько времени было в пользовательском пространстве, сколько в пространстве ядра
   * perf record + perf report - собирает профилирующие данные во время выполнения, позволяет анализировать, какие системные вызовы и функции занимают больше всего времени
5. Yandex perforator
   * Метрики: Latency, throughput, call count, user time, system time
   * Может быть добавлен в образ Yocto через bitbake-рецепт
   * Пример запуска 
     
     ```bash
     perforator record --pid <PID> --duration 60s --output profile.pb
     ```
6. sysdig
   * Метрики: latency, throughput, call count, user time, system time
   * Sysdig перехватывает системные вызовы и события ядра через модуль ядра и позволяет детально анализировать задержки, количество вызовов, ошибки и время выполнения системных вызовов
   * Через bitbake
7. bpftrace
   * Метрики: latency, throughput, call count 
   * Инструмент для динамического профилирования и трассировки системных вызовов, функций ядра и пользовательских функций с помощью eBPF
   * Можно добавить через bitbake
   * Пример запуска 
     
     ```bash
     sudo bpftrace -e 'tracepoint:syscalls:sys_enter_openat { @latency[comm] = hist(nsecs); } interval:s:10 { print(@latency); clear(@latency); }'
     ```

Связанные с IPC:
1. LTP
   * Метрики: throughput, latency
   * Включает специализированные тесты для всех основных IPC-механизмов: очередей сообщений, семафоров, общей памяти, каналов
   * Генерирует интенсивную нагрузку, проверяя корректность и производительность
   * Пример запуска 
     
     ```bash
     runltp -f ipc
     ```
2. stress-ng
   * Метрики: throughput, error rate, cpu utilization
   * Запускает стрессоры для разных IPC-механизмов: message queues (`--mq`), pipes (`--pipe`), семафоры (`--sem`), shared memory (`--shm`)
   * Создаёт высокую нагрузку, измеряет количество операций и ошибки
   * Параллельно запускает несколько процессов, интенсивно выполняющих IPC-операции
   * Позволяет варьировать количество потоков и время теста
   * Пример запуска 
     
     ```bash
     stress-ng --mq 4 --pipe 4 --sem 4 --timeout 60s
     ```
3. perf 
   * Можно с его помощью собирать статистику по выбранным событиям во время выполнения программы или теста
   * perf stat можно использовать для измерения переключений контекста, user и system time
4. Yandex perforator
   * Метрики: latency, troughput, call count, cpu utilization
   * Через bitbake
5. sysdig 
   * Метрики: latency, call count, cpu utilization, error rate
   * Позволяет отслеживать системные вызовы, связанные с IPC, измерять задержки, частоту, ошибки и нагрузку на CPU, связанную с IPC-механизмами
   * Пример запуска 
     
     ```bash
     sysdig evt.type=pipe or evt.type=shmget or evt.type=msgsnd
     ```
6. bpftrace
   * Метрики: latency, call count, throughput, cpu utilization
   * Через bitbake
   * Пример запуска 
     
     ```bash
     sudo bpftrace -e 'tracepoint:syscalls:sys_enter_pipe { @calls[comm] = count(); }'
     ```

Связанные с виртуальной памятью:
1. stress-ng
   * Метрики: Memory Throughput, Latency обработки page faults, Error rate
   * создает нагрузку на подсистему памяти, интенсивно выделяя и освобождая память, а также генерируя page faults
   * выводит статистику по количеству выполненных операций, времени и ошибкам
   * Пример запуска 
     
     ```bash
     stress-ng --vm 4 --vm-bytes 1G --timeout 60s --metrics-brief
     ```
2. vmstat
   * Метрики: Page Fault Rate (soft и hard), Swap Usage, Free Memory
   * отображает в реальном времени статистику использования памяти, swap, CPU и системных событий
   * Пример запуска - чтобы запустить vmstat с десятью обновлениями с интервалом в пять секунд, введите
     
     ```bash
     vmstat 5 10
     ```
3. perf 
   * Метрики:  Page Fault Rate, Latency обработки page faults, Context Switches
   * Пример запуска 
     
     ```bash
     perf stat -e page-faults,minor-faults,major-faults,context-switches ./my_program
     ```
4. Yandex perforator
   * Метрики: Page Fault Rate, context switches
   * Через bitbake
5. sysdig
   * page fault rate, context switches
   * Позволяет отслеживать события page faults и переключения контекста, связанные с управлением виртуальной памятью
   * Пример запуска 
     
     ```bash
     sysdig -c cswitches
     
     sysdig evt.type=page_fault_user
     ```
6. bpftrace
   * Метрики: page fault rate, context switches
   * Через bitbake
   * Пример запуска 
     
     ```bash
     sudo bpftrace -e 'tracepoint:exceptions:page_fault_user { @pf[comm] = count(); } interval:s:10 { print(@pf); clear(@pf); }'
     ```

## Определение времени работы программы в пространстве пользователя и в пространстве ядра

1. time
   * Утилита измеряет время работы программы, разделяя его на пользовательское и системное время, а также показывает общее время выполнения
   * Пример запуска
     ```bash
     time ./perl-timeout-example 100.100.100
    
     We could not ping the desired address!
    
     real    0m5.0013s
     user    0m0.004s
     sys     0m0.008s
     ```
   * Реальное: общее время от начала до конца вызова, включая время процессора, затрачиваемое на другие процессы. 
   * User: количество времени процессора, затраченного на пространство пользователя. 
   * Sys: количество процессорного времени, затраченного на пространство ядра.
2. perf
   * Пример запуска
     ```bash
     perf stat ./myprogram

           83.409183620 seconds time elapsed

           74.684747000 seconds user
            8.739217000 seconds sys
     ```

## Сбор данных в течение длительного времени 

Инструменты:
* Prometheus - система мониторинга, которая собирает метрики из различных источников и хранит их в базе данных
* Grafana - инструмент визуализации данных, который позволяет создавать графики и панели на основе данных из Prometheus
* Утилиты для тестирования (ipref, fio и др)
