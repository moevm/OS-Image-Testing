# Анализ тактов

**Анализ тактов** - это анализ производительности на уровне тактов процессора. Он может показать, сколько тактов процессора "теряется" из-за промахов кэша, операций ввода/вывода и т.д. Заключается в анализе таких величин, как количество тактов, количество выполненных инструкций, количество промахов кэша, IPC (instructions per cycle) или CPI (cycles per instruction). Применяется, чтобы определить, насколько эффективно процессор выполняет код, какие есть узкие места, где есть потенциал для оптимизации.

## Применимость к тестированию образов

Анализ тактов может быть полезен при сравнении образов ОС для оценки того, как различные компоненты ОС влияют на использование процессора.

## Инструменты для анализа тактов

* [perf](https://www.brendangregg.com/perf.html) - это инструмент для профилирования, трассировки и анализа производительности системы и приложений;
* [showboost](https://github.com/brendangregg/msr-cloud-tools/blob/master/showboost) - утилита, отображающая тактовую частоту процессора с разбивкой по интервалам;
* [turbostat](https://www.linux.org/docs/man8/turbostat.html) - это инструмент, основанный на использовании MSR (Model-specific registers), показывающий состояние процессора;
* [pmcarch](https://github.com/brendangregg/pmc-cloud-tools/blob/master/pmcarch) - это утилита, которая предоставляет общую информацию о производительности на уровне тактов процессора.
