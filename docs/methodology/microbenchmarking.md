# Микробенчмаркинг

**Микробенчмаркинг** - это измерение производительности отдельной небольшой части системы, например, функции, системного вызова.
Поскольку микробенчмаркинг предназначен для тестирования малых частей системы, этот метод характеризуется очень
высокой скоростью и низкими накладными расходами.

## Применимость к тестированию образов

В тестировании образов ОС микробенчмаркинг может быть использован для оценки производительности файловой и сетевой подсистем, опеределения скорости выполнения операций, выявления задержек.

## Инструменты

* [benchmark](https://github.com/google/benchmark) - библиотека от Google для тестирования отдельных участков кода;
* [lmbench](https://packages.debian.org/ru/sid/lmbench) - набор микробенчмарков для unix-систем. Измеряет две ключевые характеристики: задержку и пропускную способность;
* [sysbench](https://github.com/akopytov/sysbench) - набор бенчмарков для тестирования различных компонентов системы, в том числе файловой системы, процессора;
* [nanobench](https://github.com/martinus/nanobench) - еще одна C++ библиотека для микробенчмаркинга;
* [hackbench](https://www.opennet.ru/prog/info/2727.shtml) - утилита для измерения скорости создания указанного числа процессов и скорости обмена данными между ними;
* perf-bench также является инструментом микробенчмаркинга.
* [fio](https://fio.readthedocs.io/en/latest/index.html) - утилита для тестирования дисковой подсистемы, позволяющая тестировать случайное и последовательное чтение/запись, синхронные/асинхронные операции;
* [iperf](https://github.com/esnet/iperf) - клиент-серверная программа для тестирования пропускной способности сети.

Также полезные инструменты можно найти [здесь](https://github.com/topics/micro-benchmarks)
