# Kirk

[Kirk](https://github.com/linux-test-project/kirk) — официальный исполнитель тестов Linux Test Project (LTP)

## Что умеет Kirk над LTP

- Запускать LTP-сьюты локально / по SSH / в QEMU / через LTX

- Делать параллельный прогон тестов

- Прокидывать переменные окружения в сьюты 

- Имеет CLI с единым синтаксисом запуска


## Требования

- Python: ≥ 3.6

- SSH (опционально): asyncssh

- LTX (опционально): msgpack

## Текущая версия

    Текущий стабильный релиз v3.0

Все релизы можно посмотреть на [git](https://github.com/linux-test-project/kirk/releases)

## Известные баги

- Возможен некорректный код возврата 0 после прерывания сьюта

- зависание shell‑тестов примерно на 5 минут

## Что будет добавлено в будущем 

- Force quit
  
  Возможность мгновенно прервать все запущенные тесты и корректно завершить прогон с явным ненулевым статусом

- Передача переменных окружения прямо из CLI 

  Задание переменных окружения параметрами запуска

- Шардирование прогона 

  Деление большого набора тестов на несколько частей (шардов) и их параллельный запуск для уменьшения общего времени. Kirk формирует общий список тестов, после чего каждый запуск получает свой кусок по паре параметров общее число шардов и номер шарда. Каждый шард можно запускать на своём узле, а затем собрать результаты в один отчёт.

- Упрощённый режим LTP‑only

  Запуск только LTP без сторонних фреймворков и плагинов

- Интеграция ver_linux 

  Автоматический сбор версий ключевых компонент системы в отчётах для удобной диагностики

- Отказ от runtest‑файлов 
  
  Переход к выбору тестов по метаданным

- Полноценная поддержка перезагрузки

  Надёжное выполнение тестов, которые вызывают перезагрузку: автоматическое переподключение и продолжение прогона

- Возможность устанавливать тест‑фреймворк средствами kirk 

  При остановке по таймауту/сигналу прогон завершится с «красным» кодом
