# Как создать дашборд с результатами тестов в Metabase

Metabase — мощный инструмент для визуализации данных. В этом руководстве мы пройдем путь от подключения базы данных с результатами прогонов автотестов до создания удобного дашборда со сложными фильтрами.

## Шаг 1. Подключение базы данных

Для начала Metabase должен получить доступ к результатам тестовых прогонов.

1. Нажмите на иконку **шестеренки** (Settings) в правом верхнем углу и выберите **Admin settings**.
2. Перейдите во вкладку **Databases** и нажмите **Add database**.
3. Выберите PostgreSQL, введите хост, порт, название базы, логин и пароль.
4. Нажмите **Save**. Metabase запустит процесс синхронизации (это может занять пару минут).

## Шаг 2. Написание SQL-запроса (и подготовка к фильтрации)

Чтобы вывести специфичные данные (например, время выполнения тестов), мы используем сырой SQL.

1. В правом верхнем углу нажмите **+ New** (или иконку карандаша) ➔ **SQL query**.
2. Выберите базу данных, которую подключили на первом шаге.
3. Напишите запрос.
4. Нажмите синюю кнопку **Run** (Play), чтобы проверить, что данные отдаются корректно.

## Шаг 3. Настройка визуализации (Box Plot)

1. Под результатами вашего SQL-запроса нажмите кнопку **Visualization** (по умолчанию стоит таблица).
2. В появившемся меню выберите **Box plot**.
3. Зайдите в настройки графика (иконка шестеренки):
   * В поле **X-axis** выберите категорию.
   * В поле **Y-axis** выберите числовую метрику.

## Шаг 4. Сохранение графика

1. В правом верхнем углу нажмите **Save**.
2. Дайте графику понятное название.
3. Выберите коллекцию (папку) для сохранения и нажмите **Save**.

## Шаг 5. Создание и настройка дашборда

1. Если вы создаете дашборд с нуля, нажмите **+ New ➔ Dashboard**, дайте ему имя и сохраните.
2. В режиме редактирования дашборда (иконка карандаша) нажмите кнопку **+ Add question** и выберите ваш сохраненный Box plot.
3. Растяните и расположите график на сетке так, как вам удобно.

## Шаг 6. Накладываем фильтры на дашборд

1. В режиме редактирования дашборда нажмите кнопку **Add a filter** (иконка воронки).
2. Выберите тип фильтра. Например, для выбора статуса теста нажмите **Text or Category** ➔ **Dropdown**.
3. **Связывание (Маппинг):** На вашем графике появится выпадающее меню. Кликните на него и выберите переменную.
4. Нажмите **Done** на панели фильтра.
5. Повторите процесс для дат (тип фильтра **Time** ➔ **Date range**).
6. Нажмите **Save** в правом верхнем углу дашборда.

## Шаг 7. Связанные фильтры (Linked Filters)

*(Основано на [официальной документации Metabase](https://www.metabase.com/learn/metabase-basics/querying-and-dashboards/dashboards/linking-filters))*

Если у вас много фильтров (например, "Проект", "Тестовый набор" и "Тест"), вы можете связать их, чтобы выбор в одном фильтре ограничивал варианты в другом (как каскадные списки).

**Как это сделать:**

1. Перейдите в режим редактирования дашборда.
2. Нажмите на иконку **шестеренки** рядом с тем фильтром, который должен *зависеть* от другого (например, фильтр "Тестовый набор").
3. Перейдите во вкладку **Linked filters**.
4. Включите тумблер рядом с родительским фильтром (например, "Проект").
5. Теперь, если пользователь выберет проект "Альфа", в фильтре "Тестовый набор" он увидит только тесты, относящиеся к проекту "Альфа".

## Шаг 8. Лучшие практики (Best Practices) проектирования

*(Основано на [рекомендациях Metabase](https://www.metabase.com/learn/metabase-basics/querying-and-dashboards/dashboards/bi-dashboard-best-practices))*

Чтобы ваш дашборд был не просто набором графиков, а полезным инструментом:

* **Самое важное — наверх:** Читают слева направо и сверху вниз. Размещайте ключевые метрики (Number of tests, % Pass Rate) в самом верху в виде цифр (визуализация *Number*).
* **Не перегружайте:** Идеальный дашборд содержит не более 10-15 графиков. Если их больше — создайте второй дашборд и добавьте ссылки.
* **Группируйте смысловые блоки:** Используйте компонент **Text** (Markdown) на дашборде, чтобы создавать заголовки (например, `# UI Тесты` и `# API Тесты`) или оставлять инструкции для пользователей.
* **Единая цветовая схема:** Настройте цвета так, чтобы зелёный всегда значил PASS, а красный — FAIL. Это делается в настройках каждого графика.
