# Nagios

[Nagios](https://github.com/NagiosEnterprises/nagioscore) - программа по мониторингу состояния узлов сети, сервисов и серверов. Крайне надёжен и требует мало ресурсов.

Его задача заключается только в запуске команды (которая может быть чем угодно) и отслеживании кода возврата - OK, WARNING, CRITICAL или UNKNOWN. Поэтому он используется непосредственно для проверки работоспособности всех задействованных узлов.

[Руководство по установке](https://assets.nagios.com/downloads/nagioscore/docs/nagioscore/4/en/quickstart.html).

После установки и запуска сервиса становится доступен веб-интерфейс по адресу http://localhost/nagios.

## Возможности

- Мониторинг информации о текущем состоянии подключённых узлов и сервисов

- Генерация отчётов по любым отслеживаемым компонентам за заданный промежуток времени, чтобы оценить стабильность системы и существующие проблемы

- Добавление своих серверов и сервисов происходит через конфигурацию `/usr/local/nagios/etc/nagios.cfg`:

```
define host {
    use                     linux-server
    host_name               router
    address                 192.168.1.1
    max_check_attempts      4
    check_period            24x7
}

define service {
    use                     generic-service
    host_name               router
    service_description     Ping
    check_command           check_ping!100.0,20%!500.0,60%
}
```

- Уведомления о сбоях или ошибках отслеживаемых компонентов при настройке отправляются на почту, терминал или телефон

- Добавление и настройка плагинов для расширения функциональности через `/usr/lib/nagios/plugins/`

## Недостатки

- Относительно сложная установка, много команд для запуска работы сервиса

- Описание всех настроек осуществляется только через конфигурационные файлы

- Применение изменений происходит при перезагрузке сервера

- Сложности при масштабировании из-за необходимости вручную модифицировать все конфигурации

## Место хранения данных

После установки все данные будут храниться по пути `/usr/local/nagios/`.

Основные директории:

- `etc/` - хранение конфигураций

- `libexec/` - плагины проверок для сервисов, например, `check_ping` выше в документе

- `var/` - хранение всех результатов (по ним генерируются графики, отчёты)
