# Munin

[Munin](https://github.com/munin-monitoring/munin) - инструмент мониторинга подсистем узлов, предоставляющий всю собранную информацию в виде графиков.

Munin состоит из двух компонентов - `Munin Master` и `Munin Node`.

`Munin Master`

- Работает по расписанию через cron

- Запускается каждые 5 минут (можно настроить), собирает данные с узлов

- Генерирует графики и HTML страницы

`Munin Node`

- Является сервисом

- Отдает метрики когда мастер запрашивает

Низкие требования к ресурсам при небольшом количестве зависымых узлов.

[Руководство по установке](https://munin-monitoring.org/download/).

После установки и запуска Apache, а также сбора первых данных становится доступен веб-интерфейс по адресу http://localhost/munin (для стабильной версии 2.0) или http://localhost:4948 (для более новой 3.0).

## Возможности

- Мониторинг огромного числа метрик (дисков, сети, процессов, CPU, памяти, swap, температуры...) без предварительной настройки

- По результатам сбора данных с узлов, обновляет существующие графики с разными временными диапазонами (день, неделя, месяц, год)

- `Munin Master` может собирать данные с нескольких узлов

- Добавление и настройка плагинов через `/usr/share/munin/plugins`

## Недостатки

- Медленное обновление данных, не в реальном времени

- Для разных узлов - разные графики, их нельзя объединить или сравнить

- Графики нужно отсматривать вручную, чтобы найти сбои и проблемы в системе

- Довольно пустой и устаревший веб-интерфейс (только гиперссылки и графики)

- При росте числа отслеживаемых узлов возрастает нагрузка на мастера

## Место хранения данных

Munin хранит собранные данные в RRD файлах, которые находятся по пути `/var/lib/munin/localdomain/`.

После генерации графики сохраняются в виде PNG файлов в директорию `/var/cache/munin/www/localdomain/server_name/`.
